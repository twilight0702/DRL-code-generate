# 小组后续规划与分工（五人）

## 项目目标（保持简单可行）
- 完成最小可交付的代码、实验结果与报告，满足课程评分要求。
- 实验组合尽量精简：random baseline、template baseline、预热+PPO（短训练）。

## 后续规划（精简版）
1. 任务与测试整理
   - 确认 tasks/tests/env 一致，可完整跑通测试。
   - 统一实验结果输出文件（建议 results/exp_results.txt）。
2. 三条实验跑通并记录结果
   - random baseline
   - template baseline
   - 预热 + PPO（短训练，少轮）
3. 报告完成
   - 实验说明、模型说明、实验设置、结果分析、成员贡献说明。

## 五人分工（职责与交付）
成员1（组长/统筹）
- 进度统筹与提交前检查。
- 汇总实验结果与最终提交清单。
- 交付：results/exp_results.txt 与最终材料清单。

成员2（任务与测试）
- 检查 tasks/tests/env 的一致性和可复现性。
- 交付：任务说明简表与测试通过日志摘录。

成员3（模型与训练脚本）
- 整理 train.py、ppo_train.py、models/ 的最小可运行流程。
- 交付：训练日志摘要与关键超参列表。

成员4（评估与基线）
- 运行 random/template/ppo 三类实验，输出对比结果。
- 交付：对比表（成功率、平均奖励、样例生成）。

成员5（报告整合与排版）
- 汇总各成员撰写的报告部分，统一格式与行文。
- 交付：report.md 完整稿（含成员贡献）。

## 时间安排（可按剩余时间调整）
- 第1天：任务与测试整理；跑通 random/template。
- 第2天：预热 + PPO 短训练；完成结果汇总。
- 第3天：报告初稿；提交前统一检查。

## 报告分工（每人一部分）
- 成员1：实验说明（项目背景、目标、整体流程）。
- 成员2：数据集处理与实验设置（任务划分、评价指标、参数设置）。
- 成员3：模型说明（网络结构、训练流程、奖励设计）。
- 成员4：实验结果与分析（对比表、成功率、失败案例）。
- 成员5：成员贡献说明与最终排版整合。

## 提交材料对照清单
- 代码文件：完整代码、实验结果文件、数据集随代码并用相对路径索引。
- 报告文件：实验说明、模型说明、实验设置、结果分析、成员贡献说明。
